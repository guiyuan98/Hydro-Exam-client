# 毕业论文提纲

题目建议：**《面向 OI 竞赛的在线评测系统安全考试客户端设计与实现》**

## 1 绪论

- 在线评测系统在程序设计教学、训练和考试中的应用背景。
- 普通浏览器考试环境在切屏、复制粘贴、换机、换 IP 等方面的不足。
- 课题目标：基于 Hydro OJ 构建服务端，开发 Windows 考试客户端，把 OJ、编辑器、本地样例和远程判题合成一个受控考试环境。

## 2 相关技术

- Hydro OJ：题库、比赛、提交、排名、HydroJudge、沙箱和插件机制。
- Electron：桌面壳、窗口控制、Chromium Web 页面嵌入。
- MinGW/g++：本地 C++ 样例编译运行。
- 用户态监考：全屏置顶、快捷键拦截、剪贴板限制、进程检测、审计日志。

## 3 需求分析

- 教师需求：部署 OJ、创建比赛、导入题目、查看排名、处理学生锁定。
- 学生需求：进入考试、阅读题面、编辑 C++、运行样例、提交远程判题。
- 监考需求：禁止切屏、禁止复制粘贴、检测黑名单进程、锁定考试窗口。
- 安全边界：不声称绝对防作弊，强调受控机房和非内核级对抗条件。

## 4 系统设计

- 总体架构：Hydro OJ 服务端 + Windows 考试客户端 + 本地编译器。
- 服务端：Hydro 提供账号、题库、比赛、提交、判题和排名。
- 客户端：主窗口加载 Hydro，本地工具窗口提供编辑和样例运行。
- 监考扩展：后续通过 Hydro 插件记录考试策略、会话、心跳和违规事件。

## 5 系统实现

- Hydro WSL 部署与超级管理员初始化。
- 客户端配置和启动模式。
- 本地样例测评流程。
- 快捷键、剪贴板、失焦和进程黑名单实现。
- 学生发行包构建。

## 6 测试与分析

- Hydro 部署测试。
- 客户端启动与中文页面测试。
- 本地样例编译、运行、超时测试。
- 监考限制测试。
- 远程 Hydro 判题测试。

## 7 总结与展望

- 总结系统完成度和适用场景。
- 展望 Hydro 插件化监考、IP/机器绑定、远程解锁、自更新、签名和网络白名单。
